09. 提示词工程与幻觉

提示词工程的核心是通过精确输入来稳定、有效地引导模型输出目标结果。

第一部分：CROFTC 与 CO-STAR

什么是 Prompt（提示词）？

  • 定义：Prompt 是用户提供给模型的输入，用来描述任务目标。
  • 组成：通常包含 指令 / 问题 / 示例 等内容。
  • 作用：通过 Prompt 引导模型完成特定任务。

Prompting 的特点

  • 偏经验主义：需要不断尝试。
  • 模型差异大：同一 Prompt 在不同模型上表现不同。
  • 需要试错与技巧：靠大量实验与启发式规则优化。

CO-STAR 框架

  • C — Context：背景信息/约束环境
  • O — Objective：任务目标
  • S — Style：写作风格
  • T — Tone：语气
  • A — Audience：受众
  • R — Response：输出格式

CROFTC vs CO-STAR

CROFTC（角色 + 约束）：Context / Role / Objective / Form / Tone / Conditions

CO-STAR（受众 + 输出）：Context / Objective / Style / Tone / Audience / Response

对应关系：Role ≈ Style，Form ≈ Response，Conditions 可作为补充约束，Audience 在 CO-STAR 中显式强调。

写出更有效 Prompt 的技巧

  • 像和人对话一样，补充说明需求。
  • 先搭舞台再提问（角色/背景/任务）。
  • 让 AI 扮演特定身份/职业。
  • 让 AI 帮你写 Prompt。
  • 追问理由与证据，减少幻觉。
  • 反向约束（Negative Prompting）。
  • 大胆试错与重述。

提示词语气有用吗？

  • 正向礼貌不一定比中性更好。
  • 负向威胁会显著让输出变差。
  • 威胁 ChatGPT 不起作用。

第二部分：In-Context Learning

ICL 是什么？

  • ICL = few-shot prompting：在 prompt 中放少量示例。
  • 不更新参数，利用上下文引导模型行为。

ICL vs 微调

  • ICL：不改参数，即用即走。
  • 微调：训练参数，换任务需再训练。

推理时如何做到 ICL？

  • 可能是预训练涌现能力。
  • 注意力机制隐式做类似优化。
  • 通过 prompt 唤起相关模式。

第三部分：Chain of Thought / Draft

Chain of Thought（CoT）

  • 让模型先生成推理步骤再给答案。
  • 提升复杂推理正确率。

Self-consistency CoT

  • 多路径解码，多数投票。
  • 更稳更准，但成本更高。

Zero-shot CoT

  • 加一句“Let’s think step by step.”即可显著提升推理。
  • 常见触发句包括：Let’s think logically / split into steps。

Chain of Draft（CoD）

  • 只保留极简草稿，减少 token。
  • 准确率接近 CoT，成本更低。

第四部分：P-tuning

  • Prompt engineering：手工写提示词。
  • Prompt-tuning：学习可训练的软提示（PEFT）。
  • 冻结大模型参数，只训练小段 soft prompt。

第五部分：终极提示词

  • 少于 25 个词、容易记住、更有效。
  • 为推理型模型优化。

好处

  • 更清楚需求与背景。
  • 固定上下文，提升覆盖面。

第六部分：检索增强生成（RAG）

为什么需要 RAG

  • 幻觉、知识过期、来源不可靠。

核心思想

  • 检索 → 增强 → 生成。

基础流程

  • 入库：文档 → 切块 → 向量 → 索引。
  • 检索：query 向量检索 Top-K。
  • 综合：LLM 基于上下文生成答案。

评估关注点

  • Query → Context 是否相关。
  • Context → Answer 是否可支撑。

第七部分：幻觉（Hallucination）

为什么会幻觉

  • 目标是生成最可能续写，不等于真实。
  • 训练数据有错或过时。
  • 缺少真假判别器，信息稀疏时倾向猜测。

降低幻觉的方法

  • RAG 外部知识支撑。
  • 微调更事实敏感。
  • 提示词约束。
  • 让模型自检。
