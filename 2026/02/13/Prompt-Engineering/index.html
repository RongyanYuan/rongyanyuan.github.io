09. Prompt Engineering & Hallucination

Prompt engineering focuses on crafting inputs that steer model behavior reliably and efficiently toward a target outcome.

Section 1 CROFTC & CO-STAR

What is a Prompt?

  • Definition: the input you provide to describe the task goal.
  • Composition: usually includes instructions, questions, and examples.
  • Purpose: guides the model to complete a specific task.

Key characteristics

  • Experimental and empirical: requires iteration.
  • Model-dependent: same prompt varies across models.
  • Heuristics + trial-and-error are common.

CO-STAR framework

  • C — Context: background and constraints
  • O — Objective: task target
  • S — Style: writing style
  • T — Tone: attitude/emotion
  • A — Audience: intended reader
  • R — Response: output format

CROFTC vs CO-STAR

CROFTC (role + constraints): Context / Role / Objective / Form / Tone / Conditions

CO-STAR (audience + output): Context / Objective / Style / Tone / Audience / Response

Mapping: Role ≈ Style, Form ≈ Response, Conditions can be extra constraints, Audience is explicit in CO-STAR.

Tips for more effective prompts

  • Clarify intent like a conversation.
  • Set role/background/task before asking.
  • Let the model assume a specific identity or profession.
  • Ask the model to improve or generate prompts.
  • Ask for reasons/evidence to reduce hallucinations.
  • Use negative constraints (what not to output).
  • Iterate and rephrase boldly.

Does prompt style matter?

  • Polite prompting is not necessarily better than neutral.
  • Threatening prompts make results worse.
  • Threatening ChatGPT doesn’t work.

Section 2 In-Context Learning

What is ICL?

  • ICL = few-shot prompting: put a few demonstrations in the prompt.
  • No parameter updates; behavior is guided by context.

ICL vs Fine-tuning

  • ICL: no training, flexible and fast.
  • Fine-tuning: updates parameters, often requires retraining per task.

How does ICL work at inference?

  • Likely an emergent ability from pretraining.
  • Attention may act like implicit optimization.
  • Prompts help retrieve relevant patterns.

Section 3 Chain of Thought / Draft

Chain of Thought (CoT)

  • Ask the model to write intermediate reasoning steps.
  • Improves accuracy on complex reasoning tasks.

Self-consistency CoT

  • Sample multiple reasoning paths and vote.
  • More stable but costs more compute.

Zero-shot CoT

  • Add “Let’s think step by step.” to trigger reasoning.
  • Other triggers: “Let’s think logically.” / “Split into steps.”

Chain of Draft (CoD)

  • Use minimal drafts (few words per step).
  • Accuracy close to CoT with fewer tokens.

Section 4 P-tuning

  • Prompt engineering: hand-crafted discrete prompts.
  • Prompt-tuning: learn soft prompts (PEFT).
  • Freeze the LLM and train only soft prompt vectors.

Section 5 Ultimate ChatGPT Prompt

  • Short (≤25 words), memorable, more effective.
  • Optimized for reasoning models.

Benefits

  • Improves requirement understanding.
  • Forces missing context to be provided.
  • Fixes the expected output context.
  • More comprehensive answers.

Section 6 Retrieval Augmented Generation

Why RAG is needed

  • Hallucinations, outdated knowledge, unreliable sources.

Core idea (R + A + G)

  • Retrieve → Augment → Generate.

Basic RAG Pipeline

  • Ingestion: documents → chunks → embeddings → index.
  • Retrieval: query embedding → Top-K chunks.
  • Synthesis: generate answer from query + context.

Evaluation focus

  • Query → Context relevance.
  • Context → Answer support.

Section 7 Hallucination

Why hallucinations happen

  • LMs optimize likely continuation, not truth.
  • Training data can be wrong or outdated.
  • No built-in truth detector, so it guesses.

How to reduce hallucinations

  • Use RAG to ground answers.
  • Fine-tune for factual sensitivity.
  • Constrain prompts.
  • Ask the model to self-check.
