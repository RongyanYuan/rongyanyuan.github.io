06. 微调（Fine-Tuning）知识文档

微调就像给一个学识渊博的通才“攻读硕士”，让它在特定领域成为专家；不同微调策略就像不同学习方法：有的人死记硬背（SFT），有的人通过实践和反馈不断改进（RLHF/DPO），有的人挑选最优答案（拒绝采样），还有人同时跟几门课程（模型平均）……最后目标都是让模型既聪明又专业。

1. 微调的概念

  • 预训练（Pretraining）：让模型“读完互联网上所有的书”，掌握通用知识和语言能力。
  • 微调（Fine-Tuning）：让一个“博学的通才”去“攻读某个专业的硕士”，在某个特定领域或任务中成为专家。

简单理解：预训练是打好通用基础，微调是针对特定目标进行专业化训练。

2. 什么时候需要微调？

  1. 特定任务：如情感分析、医学问答、金融预测等
  2. 特定风格或语气：例如广告文案、学术写作、新闻报道
  3. 对输出可控性要求高：如生成内容的格式、逻辑或政策合规性

什么时候需要预训练？

  • 当任务需要领域知识特别强，现有模型无法覆盖时
  • 例如法律、医疗、金融领域的专业模型

3. 微调的常见策略

3.1 监督微调（Supervised Fine-Tuning, SFT）

  • 原理：用人工标注的数据训练模型，让输出符合预期
  • 应用场景：客服问答、标准化报告、特定文风写作

3.2 强化学习微调（RLHF / RLFH）

  • 原理：通过与环境交互获取奖励信号，优化模型行为
  • 流程：
    1. 收集数据（偏好数据/用户反馈）
    2. 训练奖励模型（Reward Model）
    3. 利用强化学习算法优化策略（Policy）
  • 应用场景：对话系统优化、内容生成偏好调整

3.3 直接偏好优化（Direct Preference Optimization, DPO）

  • 原理：直接用偏好数据优化模型，而不依赖 RL 步骤
  • 核心点：偏好数据收集（Preference Data Collection）至关重要
  • 应用场景：文案优化、对话回复优先级排序

3.4 拒绝采样（Rejection Sampling）

  • 原理：生成多个候选输出，按规则筛选出最符合标准的结果
  • 关键：筛选标准或规则设计非常重要，需要熟悉任务规则
  • 应用场景：广告文案生成、合规内容生成

3.5 模型平均（Model Averaging）

  • 原理：训练多个模型，平均参数形成最终模型
  • 特点：
    • 可以覆盖不同任务或领域
    • 可能导致单任务性能下降
    • 更适合生成多样化内容
  • 应用场景：内容生成、多任务生成场景

3.6 LoRA / PEFT（低秩适配 / Parameter-Efficient Fine-Tuning）

  • 原理：只微调部分参数或插入适配层，减少训练成本
  • 优点：
    • 节省计算资源
    • 可快速适配新任务
  • 应用场景：小样本任务、快速定制模型

4. 微调与预训练的关系

对比 预训练 微调
数据量 海量通用数据 特定任务/风格数据
目标 通用知识 + 语言能力 特定任务表现 + 输出风格
计算成本 高 相对低
灵活性 通用性强 专业化强
典型场景 ChatGPT 基础模型 法律问答助手、广告生成模型

总结：预训练是基础，微调是精炼。没有预训练，就没有微调的土壤；没有微调，再强大的基础模型也可能达不到专业需求。

微调（Fine-Tuning）策略可视化

5. 微调应用场景示例

  • 客服与问答系统：SFT + RLHF 调整回复风格与准确性
  • 文案生成：DPO + 拒绝采样优化偏好输出
  • 多任务生成：模型平均实现不同生成风格的平衡
  • 快速任务适配：LoRA/PEFT 在小样本任务中快速微调
  • 行业专业模型：结合领域预训练 + 微调，生成高专业度内容（如医疗、金融、法律）

6. 风趣理解

可以把微调理解为给通才安排一个更聚焦的学习路径。
