07. RAG / MCP / A2A
Feb 11 2026

Session 5: Product Workflow and RAG System Design

1. Product workflow overview

1.1 Workflow + data pipeline

A smart product workflow typically includes:

  1. User input → handled by an agent or multi-agent collaboration
  2. Memory module analyzes inputs and decides what to keep
    • keep memory minimal but high precision
    • consider cost (tokens, GPU, engineering)
  3. Call tool APIs (MCP) or other agents (A2A) to solve tasks
    • MCP (Model + Tool/Prompt + RAG)
    • A2A (Agent-to-Agent, like asking a colleague)
  4. Polishing agent refines the output based on user habits and preferences, then renders back to the user

Core idea: the workflow is not just Q&A, but multi-module collaboration + memory management + output optimization.

2. Cost and efficiency considerations

  • Cost accumulation: unlike SaaS with decreasing marginal cost, each query consumes tokens, compute, and engineering time.
  • Memory optimization: minimal-but-precise memory lowers cost and improves performance.
  • Multi-agent collaboration: assign tasks based on each model’s strengths.

3. RAG system design (Retrieval-Augmented Generation)

Core goal: high-quality answers, complete context, and controlled cost.

3.1 Data preparation

  1. Data cleaning
    • documents can be messy, overly detailed, or contradictory
    • overly detailed content hurts chunking; split by text type
    • vector storage: map text → embedding space for semantic accuracy
  2. Data chunking
    • split documents into suitable chunks for indexing
    • consider context limits (e.g., 128kb); too long or too short hurts understanding
    • PM + algorithm team co-design chunking rules
  3. Vectorization
    • strong embedding strategy + retrieval strategy
    • like dictionary lookup: “how much is this pot?” → find price tags → rank/filter
    • multi-recall strategies:
      • split query dimensions (price, warranty, promotions)
      • generate multiple queries for parallel retrieval
      • reweight results by context (e.g., price-sensitive users get cheaper items)
    • can combine with fine-tuning data to optimize preference outputs

3.2 Operations and memory management

  1. Context management: define what to remember (name, age, preferences)
  2. Answer strategy: decide tone and style based on business context
  3. High-frequency optimization:
    • pre-generate cache for common questions
    • prepare summary answers to cut retrieval cost and latency

4. Multi-agent collaboration

  • A2A (Agent-to-Agent): specialized agents collaborate like colleagues
  • Multi-model task assignment: allocate work by model capability
  • Goal: higher efficiency, lower single-model load, better accuracy and response speed

5. Key points summary

  1. Memory optimization: minimal and precise memory improves quality and lowers cost
  2. Cost factors: token usage, GPU compute, and iteration costs accumulate
  3. Multi-agent coordination: assign tasks by strengths for complementary coverage
  4. Data quality is core: cleaning, chunking, and vectorization underpin good RAG
  5. Pre-handle frequent queries: cache answers and summaries to reduce repeats

One-line summary: RAG + multi-agent collaboration = efficient, accurate, and controllable intelligent Q&A.

6. Fun analogy

The system is like a smart office:
- MCP agent is the tool assistant that fetches materials
- A2A agent is asking colleagues to collaborate
- Polishing agent is the secretary who tailors the output style
- Memory module is the notebook that keeps only the essentials
- RAG processing is the filing system that helps everyone find answers quickly
- Cost and efficiency are the office budget that must be managed carefully
