08. AI Training and Fine-Tuning Strategy Summary

1. Pretraining

  • What it does: train a brand‑new model from scratch on massive unlabeled text to learn general language and world knowledge.
  • When to use:
    • building a brand‑new model architecture from the ground up.
    • owning a huge, unique corpus that existing models don’t cover.
  • Key points:
    • extremely expensive in compute and data.
    • mostly done by big companies and research institutions.
    • most developers will never need to pretrain.

2. Fine‑Tuning

  • What it does: continue training on task-specific data to adapt a pretrained model.
  • When to use:
    • the model must learn domain terms, formats, or logic (law, medicine, programming).
    • the model must follow specific output style or instructions.
    • the task is clear and you have hundreds to thousands of high‑quality pairs.
  • Representative techniques: full fine‑tuning, LoRA / QLoRA.

3. Supervised Fine‑Tuning (SFT)

  • What it is: the main form of fine‑tuning using question‑answer pairs.
  • When to use:
    • when you have clear, correct input‑output pairs to teach a task quickly.
    • it’s the first step in most fine‑tuning pipelines.
    • Example: teach the model to map user queries to API parameters.

4. Alignment: DPO and RLHF

  • What it is: alignment methods to match human preferences (helpful, safe, concise, logical).
  • When to use:
    • after SFT, when the model can do the task but not well enough.
    • when there is no single correct answer, but you can judge better answers.
  • Recommendation:
    • RLHF: mature but complex, unstable, resource‑heavy.
    • DPO: simpler, stable, popular, often comparable to RLHF.

5. Retrieval‑Augmented Generation (RAG)

  • What it does: retrieve external docs and feed them with the query to generate answers.
  • When to use:
    • knowledge changes frequently (news, prices).
    • private or internal data is required.
    • data is too large to inject via fine‑tuning.
    • answers must be source‑grounded.
  • Core advantages: low update cost and reduced hallucination risk.

6. Distillation / pruning and related methods

  • What it does: transfer knowledge from a large teacher model to a smaller student.
  • When to use:
    • resource‑constrained deployment (mobile, edge).
    • reduce API costs by using smaller models.
    • faster inference and higher throughput.
  • Tradeoff: student models usually underperform the teacher.

7. Re‑selecting the base model

  • understand strengths and weaknesses of base models.
  • know capability boundaries.
  • be clear about current problems.

8. Strategy implementation steps

  1. Quickly identify issues: test with low‑cost prompts.
  2. Choose optimization methods: fine‑tuning, RAG, distillation, etc.
  3. Final goal: improve user experience and solve real tasks.

Summary

  • Pretraining: build a general “brain,” extremely costly.
  • Fine‑tuning: specialize via SFT, DPO, RLHF, LoRA.
  • RAG: external knowledge, easy to update.
  • Distillation/pruning: trade accuracy for efficiency.
  • Strategy: identify issues → choose method → improve experience.
