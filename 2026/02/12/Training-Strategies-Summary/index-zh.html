08. AI 模型训练与微调策略知识文档

1. 预训练（Pretraining）

  • 做什么：从零开始，在海量无标注文本上训练一个全新模型，学习通用语言规律和世界知识。
  • 何时用：
    • 从头创建一个全新的模型架构时。
    • 拥有一个极其庞大且独特的文本数据集，且现有模型无法覆盖时。
  • 关键点：
    • 计算和数据代价极其高昂。
    • 只有巨头公司和研究机构会做。
    • 绝大多数开发者永远不需要进行预训练。

2. 微调（Fine-Tuning）

  • 做什么：在预训练好的基础模型上，用特定任务数据继续训练，调整其参数以适应新任务。
  • 何时用：
    • 模型需要掌握特定领域术语、格式和逻辑（如法律、医疗、编程）。
    • 需要模型遵循特定输出风格或指令（如固定格式写邮件、生成某种文风）。
    • 任务明确且有几百到几千条高质量输入-输出配对数据。
  • 代表技术：全参数微调、LoRA / QLoRA（参数高效微调）。

3. 监督微调（SFT, Supervised Fine-Tuning）

  • 是什么：SFT 是微调的最主要形式，通过“问题-标准答案”对监督训练模型。
  • 何时用：
    • 当你有清晰、正确的“问题-标准答案”对时，用于快速教会模型完成某项任务。
    • 它是微调 Pipeline 的第一步和基础。
    • 示例：教模型把用户查询转换为 API 调用参数。

4. 对齐微调：DPO 与 RLHF

  • 是什么：都属于对齐技术，目标是让输出更符合人类偏好（更有帮助、更无害、更简洁）。
  • 何时用：
    • SFT 之后，当模型“会做”但“做不好”。
    • 没有唯一正确答案，但可以判断哪个更好。
  • 选择建议：
    • RLHF：流程复杂、训练不稳定、资源消耗高，适合资源充足团队。
    • DPO：更直接稳定，当前更流行，通常建议优先尝试。

5. 检索增强生成（RAG, Retrieval-Augmented Generation）

  • 做什么：不改变模型本身，而是从外部知识库检索相关文档，连同问题一起输入模型生成答案。
  • 何时用：
    • 知识需要实时更新（新闻、股价）。
    • 涉及私有、未公开数据（公司内部文档、个人笔记）。
    • 数据量太大无法通过微调注入（整本手册、产品目录）。
    • 需要溯源和出处。
  • 核心优势：知识更新成本低，减少幻觉风险。

6. 模型蒸馏 / 剪枝等技术

  • 做什么：把大型教师模型的能力迁移到小型学生模型中。
  • 何时用：
    • 部署资源受限（手机、边缘设备）。
    • 降低 API 调用成本。
    • 提升推理速度和并发能力。
  • 注意：学生模型性能通常低于教师模型。

7. 是否要重新选择底座模型

  • 了解不同基座模型的优缺点
  • 明确模型能力边界
  • 清楚当前遇到的问题

8. 策略实施步骤

  1. 快速识别问题：用低成本 Prompt 测试模型短板。
  2. 选择优化手段：根据测试结果选微调、RAG 或蒸馏等。
  3. 最终目标：提升用户体验并解决实际任务问题。

总结

  • 预训练：打造通用“大脑”，成本极高。
  • 微调：通过 SFT、DPO、RLHF、LoRA 等方式专业化。
  • RAG：外部文档辅助生成，知识易更新。
  • 蒸馏/剪枝：用效率换成本。
  • 策略实施：识别问题 → 选择手段 → 提升体验。
